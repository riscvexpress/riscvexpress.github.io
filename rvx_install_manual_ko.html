RISC-V eXpress Install Manual

한규승, 이석호, 장형욱, 김상필, 이재진

개요

  • RISC-V eXpress (이하 RVX) 는 RISC-V 환경을 활용하여 MCU 급의 저전력 SoC를 개발하기 위한 HW/SW development kit 이다.

    • 논문제목: Developing TEI-Aware Ultralow-Power SoC Platforms for IoT End Nodes

    • 게재지: IEEE Internet of Things Journal, vol. 8, no. 6, 2021

    • https://ieeexplore.ieee.org/document/9208740

  • 본 메뉴얼은 rvx mini git을 기반으로 사용자의 컴퓨터에 RVX를 설치하기 위한 메뉴얼이다.

주의사항

  • RVX를 이용하여 만든 모든 결과물은,

    • 특정 수업 또는 연구과제 등 사전에 지정된 목적과 범위를 벗어나서 활용할 수 없다.

    • 지정된 사용자 또는 사용기관 이외에 제3자에게 배포할 수 없다.

    • 상업적 목적으로 사용하려면 기술이전이 필요하다.

  • ETRI에서 제공하는 RVX server을 사용할 경우,

    • RVX mini를 통한 제공된 기능만을 이용해야한다.

    • 이를 의도적으로 위반하여 사고가 발생하면 사용자가 비용을 포함한 모든 책임을 진다.

준비물

  • 라이센스 툴

    1. RTL 시뮬레이터 = Modelsim/Questa or Xcelium/NCSim/Incisive

    2. FPGA 툴 = Xilinx Vivado 2021.2 이상

  • 성능이 좋은 컴퓨터

    1. 라이센스 툴들을 돌리기 위해서는 고성능이 필요하다.

    2. 실제로 필요한 성능은 각 라이센스 툴의 요구사항을 참조한다.

    3. 참고로, RVX 자체 엔진은 고성능을 요구하지 않는다.

  • 아래 해당하는 OS 중 라이센스 툴이 안정적으로 돌아가는 버전 설치 권장

    1. CentOS 7, 8

    2. Redhat 8, 9

    3. Ubuntu 18, 20, 22, 24 (20 이상 권장)

    4. Windows 10 (Home, Enterprise)

    5. Windows 11 Home

  • Linux들의 경우, 모든 locale 설정값들이 en_US.UTF-8 또는 C.UTF-8 이어야한다.

    1. GUI를 이용해서 설정할 경우, language의 formats를 United States로 설정한다.

    2. 또는, /etc/ssh/ssh_config 파일에서 SendEnv LANG LC* 줄 앞에 #을 붙여 주석처리한다.

  • 컴퓨터의 관리자 권한이 있는 계정

    1. Linux는 sudo 권한, Windows는 관리자 권한이 있어야한다.

  • RVX git 주소

    1. 교수 또는 프로젝트 관리자 등에게 받도록 한다.

  • RVX server의 IP 주소, 계정, 비밀번호

    1. 교수 또는 프로젝트 관리자 등에게 받도록 한다.

    2. 사용할 컴퓨터 및 컴퓨터 계정마다 하나씩 필요하다.

알아두어야 할 사항

  • RVX git repository가 설치된 위치를 부른다.

  • 메뉴얼에서 알려주는 명령 중에서 #로 시작되는 부분은 상황에 맞게 바꿔서 입력해야한다.

  • Linux에서 명령어창은 bash shell을 사용한다.

  • Linux에서 특정 디렉토리를 PATH 변수에 추가하는 방법

    1. .bashrc 파일에 "export PATH=$PATH:#(특정디렉토리)"를 추가한다.

  • Windows에서 명령어창은 [Windows Power Shell]을 사용한다.

  • Windows에서 [Windows Power Shell]를 실행하는 방법

    1. 검색에서 power를 검색한다.

    2. [Windows Power Shell]를 클릭한다.

  • Windows에서 특정 디렉토리 위치에서 명령어창을 실행하는 방법

    1. Windows 탐색기에서 원하는 디렉토리로 이동한다.

    2. 오른쪽 파일뷰에서 쉬프트키를 누른 상태로 마우스 오른쪽 클릭을 한다.

    3. [여기서 PowerShell 창 열기]를 클릭한다.

  • Windows에서 환경변수를 설정하는 방법

    1. [시스템 속성]을 연다.

      • 검색창에서 SystemPropertiesAdvanced.exe를 검색해서 실행한다.

      • 또는, ${RVX_MINI_HOME}\install\open_system_properties.bat를 실행한다.

      • 또는, 명령어창에서 SystemPropertiesAdvanced.exe를 실행한다.

    2. [고급]탭에서 [환경변수] 클릭

    3. 시스템 변수가 아니라 사용자 변수에 추가한다.

    4. 추가하려는 변수가 이미 존재하면 해당 변수를 클릭한 후 [편집]을 한다.

    5. 추가하려는 변수가 존재하지 않으면 [새로 만들기]를 한다.

    6. 환경변수를 수정하고 나서는 기존의 명령어창을 껐다가 다시 켜야한다.

      [Install Driver]를 누른다.

    4. [Olimex OpenOCD JTAG ARM-USB-TINY-H (Interface 1)] 룰 선택하고
      [Install Driver]를 누른다.

동기화

  • RVX 서버에서 구동환경을 가져오는 작업이다.

    • Git을 처음 받았을 때는 반드시 한번 실행해야한다.

    • 이후에는 update 과정에서 자동으로 실행된다.

    • 동기화가 끝나면 최신 버전의 메뉴얼들이 ${RVX_MINI_HOME}/manual에 생긴다.

  • 동기화 과정 중에 RVX 서버의 계정정보를 입력하게 된다.

    • 계정정보는 미리 알고 있어야한다.

    • ${RVX_MINI_HOME}/.rvx_server_config에 저장된다.

    • RVX 계정정보를 변경하고 싶은 경우 해당 파일을 삭제한다.

    1. 이동한다.

    2. make sync

      • SSH 접속 여부는 no로 대답한다.

계정 초기화 및 동기화

  • sync 과정에서 문제가 생길 때 실행한다.

    1. 이동한다.

    2. make resync

자동 Update

  • 사용 중에 Update하라는 메시지가 뜨면 진행한다.

    1. 작업한 내용이 있다면 platform 디렉토리를 다른 곳으로 백업한다.

    2. 이동한다.

    1. ./update.sh

      • update.sh 파일이 없으면 make config

    2. ./rvx_setup.sh 파일이 새로 생겼으므로 명령어창을 새로 켜서 작업한다.

    1. update.bat

      • update.bat 파일이 없으면 make config

    2. 환경변수가 업데이트 되었으므로 명령어창을 새로 켜서 작업한다.

    3. 플랫폼을 clean을 하고 syn부터 다시 작업한다.

수동 Update

  • 관리자의 요청에 의해 진행한다.

    1. 이동한다.

    2. 작업한 내용이 있다면 platform 디렉토리를 다른 곳으로 백업한다.

    3. git checkout .

    4. git pull origin master

    5. git submodule init

    6. git submodule update

    7. make reconfig_python

    8. make pip3

    9. make config

    10. 명령어창을 새로 켜서 작업한다.

    11. make sync

    12. 플랫폼을 clean을 하고 syn부터 다시 작업한다.
